# Módulo de Contexto Rodoviário (MCR)
**Fonte de Dados:** Camada de Rodovias Federais (ID: 34) - DNIT/SNV

## 1. Visão Geral
Este módulo descreve a integração dos metadados georreferenciados do **Sistema Nacional de Viação (SNV/DNIT)** ao sistema de rastreamento baseado em Google Maps. A estratégia foca no enriquecimento de dados (backlog informativo).

## 2. Estrutura de Atributos Críticos
| Campo SNV    | Nome Amigável | Descrição / Uso                             |
| :----------- | :------------ | :------------------------------------------ |
| `vl_br`      | Rodovia       | Identificação da rodovia (ex: 010, 222).    |
| `vl_km_inic` | KM Inicial    | Marco zero ou inicial do trecho.            |
| `vl_km_fina` | KM Final      | Limite final do trecho.                     |
| `ds_superfi` | Pavimentação  | PAVIMENTADA, NÃO PAVIMENTADA ou PLANEJADA.  |
| `ds_jurisdi` | Jurisdição    | Regime de administração (DNIT ou Delegada). |

## 3. Lógica de Integração (Backend)

### 3.1. Processo de Vinculação (Matching)
1. **Trigger:** O veículo envia as coordenadas coordenada `(lat, lng)`.
2. **Buffer Geográfico:** O sistema consulta a base do DNIT buscando a polilinha mais próxima (raio de 50m).
3. **Interpolação Linear:** Com base nos campos de KM, o sistema estima o **KM Atual** do veículo.

### 3.2. Exemplo de Payload de Contexto

```json
{
  "veiculo_id": "19001",
  "contexto_rodoviario": {
    "via": "BR-010",
    "km_estimado": 254.8,
    "uf": "MA",
    "condicao": "PAVIMENTADA",
    "administracao": "DNIT"
  }
}
```

## 4. Interface do Usuário (UI/UX)
* **Pop-up do Veículo:** Exibir `Localização: BR-010, KM 254.8 (MA)`.
* **Alertas:** Se `ds_superfi` for "NÃO PAVIMENTADA", emitir aviso de risco de atraso.

## 6. Lógica de Humanização de Localização (Campos ds_local)

Para tornar o sistema intuitivo para operadores de tráfego, o MCR não deve exibir apenas dados brutos. A descrição do trecho deve ser processada por uma lógica de concatenação condicional para evitar campos vazios e garantir clareza semântica.

### 6.1. Algoritmo de Tratamento de Strings
A exibição da referência seguirá a seguinte hierarquia de prioridade:

| Condição                   | Formato de Saída                    | Exemplo Prático                     |
| :------------------------- | :---------------------------------- | :---------------------------------- |
| **Início e Fim presentes** | `Entre [ds_local_i] e [ds_local_f]` | "Entre Entr. MA-122 e Entr. BR-226" |
| **Apenas Início presente** | `Próximo de [ds_local_i]`           | "Próximo de Entr. MA-122"           |
| **Apenas Fim presente**    | `Próximo em [ds_local_f]`           | "Próximo de Entr. BR-226"           |
| **Ambos ausentes**         | `Referência não identificada`       | "Referência não identificada"       |

### 6.2. Implementação Sugerida (Pseudo-código)
Esta lógica deve ser aplicada antes da renderização do Pop-up no mapa:

```javascript
/**
 * Trata as descrições de trecho do DNIT para exibição amigável
 */
function getDescricaoTrecho(ds_local_i, ds_local_f) {
  const i = ds_local_i?.trim();
  const f = ds_local_f?.trim();

  if (i && f) return `Entre ${i} e ${f}`;
  if (i) return `Próximo de ${i}`;
  if (f) return `Próximo de ${f}`;
  
  return "Referência não identificada";
}
